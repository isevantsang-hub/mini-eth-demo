# 如何生成一个相同基本目录结构的新项目

有两种常用方式，任选其一即可。

---

## 方式一：用当前项目当模板（复制整份再改名）

适合：想立刻得到一个能跑的项目，再改名字和合约内容。

1. **复制整个项目文件夹**  
   例如复制 `mini-eth-token` 为 `my-new-token`（或在别的路径下复制一份）。

2. **删掉依赖和构建产物**（可选，便于干净安装）：
   ```bash
   cd my-new-token
   rm -rf node_modules package-lock.json
   ```
   若不需要保留原 lockfile，可删 `package-lock.json`；保留则跳过删除它。

3. **安装依赖**：
   ```bash
   npm install
   ```

4. **改项目名**（可选）：  
   打开 `package.json`，把 `"name": "mini-eth-token"` 改成 `"name": "my-new-token"`。

5. **按需删改合约和脚本**  
   在 `contracts/`、`scripts/`、`test/`、`ignition/` 里改或新增文件即可。

---

## 方式二：用脚手架脚本生成全新目录

适合：想在**空目录**里生成一套和当前项目一样的基本结构，不依赖“复制整份项目”。

当前项目里已经带了一个脚本，可以在**指定目录**下生成相同结构（含最小示例合约、测试、部署、配置）。

### 怎么用

在**本仓库根目录**（`mini-eth-token`）下执行：

```bash
node scripts/scaffold-new-project.mjs <目标路径>
```

- `<目标路径>` 可以是相对路径或绝对路径。
- 若目标目录不存在会自动创建；若已存在，只会在里面创建/覆盖脚本生成的文件，不会删你已有文件。

**示例**：

```bash
# 在上级目录生成新项目 my-new-token
node scripts/scaffold-new-project.mjs ../my-new-token

# 在当前目录下生成子文件夹 new-project
node scripts/scaffold-new-project.mjs ./new-project
```

### 生成后你要做的事

1. 进入生成好的目录并安装依赖：
   ```bash
   cd ../my-new-token
   npm install
   ```
2. 运行一次快速验证：
   ```bash
   npm run quickstart
   ```

之后就可以在这个新目录里正常开发：改合约、加测试、改部署模块等。

---

## 生成出来的基本目录结构（两种方式一致）

```
项目根/
├── contracts/           # Solidity 合约与 .t.sol 单测
├── scripts/             # Hardhat 脚本
├── test/                # TypeScript 集成测试
├── ignition/
│   └── modules/         # Ignition 部署模块
├── hardhat.config.ts
├── tsconfig.json
├── package.json
├── .gitignore
└── README.md
```

用方式一 = 当前这份目录原样复制后改名、删 node_modules 再 `npm install`。  
用方式二 = 运行 `node scripts/scaffold-new-project.mjs <目标路径>` 在目标路径下生成上述结构，再进入该目录 `npm install` 即可开始开发。
