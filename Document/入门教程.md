# 最基础入门：编写 → 测试 → 部署

按下面三步做一遍，你就能跑通「改合约 → 跑测试 → 本地部署」的完整流程。

---

## 第一步：编写合约

项目里已经有一个最简单的合约：**计数器** `contracts/Counter.sol`。

- 打开 `contracts/Counter.sol`，你会看到：
  - 一个状态变量 `x`
  - 两个函数：`inc()`（加 1）和 `incBy(uint by)`（加 by）

**你可以先改一点试试**：比如把 `emit Increment(1)` 改成 `emit Increment(2)`，保存后去第二步跑测试，看测试会不会失败（体验「测试在保护你的逻辑」）。

改完如果不想保留，改回 `emit Increment(1)` 即可。

---

## 第二步：测试

在项目根目录打开终端，执行：

```bash
npm test
```

会做两件事：

1. **Solidity 测试**：运行 `contracts/Counter.t.sol` 里的 Foundry 风格单测。
2. **TypeScript 测试**：运行 `test/Counter.ts`，用 viem 部署 Counter 并检查事件、数值。

看到一列通过的用例就说明当前合约通过测试。

---

## 第三步：部署（本地）

在项目根目录执行：

```bash
npm run deploy:local
```

会在本地模拟一条链，用 Ignition 部署 `Counter` 合约，并执行部署模块里写好的那次 `incBy(5)` 调用。终端里会打印部署结果和合约地址。

---

## 一键跑完全流程（推荐先试这个）

想一次性体验「编译 → 测试 → 本地部署」，在根目录执行：

```bash
npm run quickstart
```

会按顺序执行：编译合约 → 跑全部测试 → 本地部署。适合第一次跑通整个项目。

> **提示**：第一次运行 `compile` 或 `quickstart` 时，Hardhat 可能会联网下载 Solidity 编译器，需要保持网络畅通。

---

## 常用命令速查

| 你想做的事       | 命令 |
|------------------|------|
| 只编译合约       | `npm run compile` |
| 跑全部测试       | `npm test` |
| 只跑 Solidity 测试 | `npm run test:solidity` |
| 只跑 TS 测试     | `npm run test:node` |
| 本地部署 Counter | `npm run deploy:local` |
| 一键编译+测试+部署 | `npm run quickstart` |

---

接下来你可以：在 `contracts/` 里加新合约，在 `test/` 里加对应测试，在 `ignition/modules/` 里加部署模块，然后用上面命令编译、测试、部署。
